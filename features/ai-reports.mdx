Kairo's report generator produces executive summaries that give you a structured view of a Coder's situation, your clan's overall health, or the entire cohort. Reports are generated on demand from the TL dashboard and stored in the `ai_reports` table.

## Report target types

Every report targets one of three scopes, defined by the `report_target_enum`:

<CardGroup cols={3}>
  <Card title="coder" icon="user">
    A focused report on a single Coder. Includes their risk level, academic progress, soft skills snapshot, and AI recommendations for how you can support them.
  </Card>
  <Card title="clan" icon="users">
    An aggregate report for all Coders in your clan. Highlights the clan average score, distribution of risk levels, common struggling topics, and patterns in learning styles.
  </Card>
  <Card title="cohort" icon="building">
    A cross-clan executive report covering all Coders in the current bootcamp cohort. Useful for program-wide analysis and reporting to Riwi leadership.
  </Card>
</CardGroup>

## What a report contains

Every report — regardless of target type — is stored in `ai_reports` with the following structure:

```sql
CREATE TABLE ai_reports (
  id             SERIAL PRIMARY KEY,
  target_type    report_target_enum NOT NULL,  -- 'coder', 'clan', 'cohort'
  target_id      INT NOT NULL,
  summary_text   TEXT NOT NULL,
  risk_level     risk_level_enum,              -- 'low', 'medium', 'high'
  recommendations TEXT,
  generated_at   TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  viewed_by_tl   BOOLEAN DEFAULT FALSE
);
```

| Field | Description |
|-------|-------------|
| `target_type` | The scope of the report: `coder`, `clan`, or `cohort` |
| `target_id` | The ID of the Coder, clan, or cohort the report covers |
| `summary_text` | The AI-generated narrative summary of the situation |
| `risk_level` | The overall risk classification for this target at generation time |
| `recommendations` | Concrete next steps the AI recommends for you as the TL |
| `viewed_by_tl` | `false` until you open the report; set to `true` on first view |

## Generating a report

<Steps>
  <Step title="Select a Coder (for coder reports)">
    Click a Coder's row in the dashboard table to open their detail panel on the right side of the screen.
  </Step>
  <Step title="Click the PDF button">
    In the detail panel's action row, click the **PDF** button (the `fa-file-pdf` icon next to the **Enviar** feedback button). This sends a request to the Python AI microservice at `POST /generate-report`.
  </Step>
  <Step title="Wait for generation">
    The AI service reads the Coder's `soft_skills_assessment`, `moodle_progress`, active `risk_flags`, and `complementary_plans` data. Generation typically takes 5–15 seconds.
  </Step>
  <Step title="View the report">
    Once generated, the report appears in the reports panel. Opening it sets `viewed_by_tl = true` on the report record, clearing the new-report notification.
  </Step>
</Steps>

<Info>
  For clan and cohort reports, use the **Reports** section available from the dashboard navigation rather than the Coder detail panel.
</Info>

## The Python `/generate-report` endpoint

Report generation is handled by a dedicated Python AI microservice, not the Node.js backend. When you trigger a report, the Node backend forwards the request to:

```
POST /generate-report
```

The service receives the target type and target ID, assembles the relevant context from the database, calls the language model, and writes the completed `ai_reports` row back to PostgreSQL.

The `ai_generation_log` table records every call to this service — including the input payload, output payload, model name, execution time in milliseconds, and whether the call succeeded — for full auditability.

## Report notifications

The `viewed_by_tl` flag drives the new-report notification flow:

- When a report finishes generating, `viewed_by_tl` is set to `false`.
- The notification bell in the topbar shows a red dot until you open the report.
- The moment you view the report, Kairo sets `viewed_by_tl = true` and the dot disappears.

<Tip>
  Reports are not deleted when viewed. You can revisit any past report from the reports list to track how a Coder's situation evolved over time.
</Tip>
