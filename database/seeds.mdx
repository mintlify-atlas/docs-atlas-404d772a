The `database/seeds/` directory contains SQL scripts that populate the database with test data for development and QA. Do not run seeds against a production database.

```
database/seeds/
├── users.sql    # Users, soft-skills assessments, modules, topics, progress, struggling topics
└── README.md
```

---

## Running the seeds

<Tabs>
  <Tab title="Supabase SQL Editor">
    1. Open your Supabase project dashboard.
    2. Navigate to **SQL Editor**.
    3. Paste the full contents of `seeds/users.sql`.
    4. Click **Run**.
  </Tab>
  <Tab title="Terminal (psql)">
    ```bash
    psql -U your_user -d your_db -f seeds/users.sql
    ```
  </Tab>
</Tabs>

The script runs inside a `BEGIN` / `COMMIT` transaction. If any statement fails, the entire seed is rolled back.

<Warning>
The seed file includes a commented-out `TRUNCATE TABLE users RESTART IDENTITY CASCADE;` line. If you uncomment it, all existing users and every row that cascades from them (plans, progress, feedback, etc.) will be permanently deleted before the new data is inserted.
</Warning>

## Tables not seeded

The following tables are left empty and populated dynamically during normal application use:

| Table | Populated when |
|---|---|
| `complementary_plans` | TL triggers AI plan generation |
| `plan_activities` | AI writes activities when generating a plan |
| `activity_progress` | Coder marks an activity complete |
| `tl_feedback` | TL sends a feedback message |
| `risk_flags` | AI risk detector raises an alert |
| `ai_reports` | TL requests a report |
| `ai_generation_log` | Any AI call completes |

## Verifying the seed

Run these queries in the Supabase SQL Editor after applying the seed to confirm everything loaded correctly:

```sql
SELECT COUNT(*) FROM users;                   -- Expected: 8
SELECT COUNT(*) FROM modules;                 -- Expected: 6
SELECT COUNT(*) FROM soft_skills_assessment;  -- Expected: 6
SELECT COUNT(*) FROM topics;                  -- Expected: 12
SELECT COUNT(*) FROM moodle_progress;         -- Expected: 6
SELECT COUNT(*) FROM coder_struggling_topics; -- Expected: 6
```

## Resetting seed data

```sql
-- Truncate from the top of the dependency tree; CASCADE handles children
TRUNCATE TABLE users RESTART IDENTITY CASCADE;
-- Then re-run seeds/users.sql
```

If you need to reset specific tables without losing user accounts:

```sql
TRUNCATE TABLE moodle_progress CASCADE;
TRUNCATE TABLE coder_struggling_topics CASCADE;
```
