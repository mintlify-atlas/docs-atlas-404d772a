This guide walks you through cloning the repository, configuring both backends, setting up the database, and verifying that all three services are running together.

## Prerequisites

Before you begin, make sure you have the following installed:

- **Node.js 18+** — [nodejs.org](https://nodejs.org)
- **Python 3.11+** — [python.org](https://python.org)
- **A Supabase account** — [supabase.com](https://supabase.com) (free tier works)
- **Git**

---

<Steps>

<Step title="Clone the repository">

```bash
git clone https://github.com/Riwi-Project-C6/Learning-Platform.git
cd Learning-Platform
```

The repository contains four top-level directories: `backend-node`, `backend-python`, `frontend`, and `database`.

</Step>

<Step title="Create a Supabase project">

1. Go to [supabase.com](https://supabase.com) and create a new project.
2. Wait for the database to provision (roughly one minute).
3. In your project dashboard, navigate to **Settings → Database** and copy the **Connection string** (URI format). You'll need this for `DATABASE_URL`.
4. Navigate to **Settings → API** and copy:
   - **Project URL** → `SUPABASE_URL`
   - **anon / public key** → `SUPABASE_ANON_KEY`
   - **service_role key** → `SUPABASE_SERVICE_KEY` (for the Python backend)

</Step>

<Step title="Run the database schema">

In the Supabase dashboard, open **SQL Editor** and run the contents of `database/schema.sql`. This creates all 14 tables, 8 ENUM types, and 2 views.

```bash
# Alternatively, run from the terminal with psql:
psql "your_database_url" -f database/schema.sql
```

Then apply Row Level Security policies:

```bash
psql "your_database_url" -f database/rls_policies.sql
```

<Note>
Optionally load sample data for development by running `database/seeds/users.sql`. This creates 8 users (2 Team Leaders, 6 Coders across three clans) with soft-skills assessments and Moodle progress records.
</Note>

</Step>

<Step title="Configure backend-node environment">

Create the file `backend-node/.env` with the following variables:

```env
# SERVER CONFIGURATION
PORT=3000
NODE_ENV=development

# SECURITY & SESSIONS
SESSION_SECRET=your_session_secret

# DATABASE (POSTGRESQL - SUPABASE)
DB_HOST=your_host
DB_PORT=5432
DB_NAME=your_name
DB_USER=postgres.your_user
DB_PASSWORD=your_password
DATABASE_URL=your_database_url

# SUPABASE CONFIG
SUPABASE_URL=your_supabase_url
SUPABASE_ANON_KEY=your_anon_key

# FRONTEND CONNECTION
FRONTEND_URL=http://127.0.0.1:5500/frontend

# PYTHON CONNECTION
PYTHON_API_URL=http://localhost:8000

# SOCIAL AUTH - GITHUB
GITHUB_CLIENT_ID=your_client
GITHUB_CLIENT_SECRET=your_client_secret

# SOCIAL AUTH - GOOGLE
GOOGLE_CLIENT_ID=your_client
GOOGLE_CLIENT_SECRET=your_client_secret

# AUTHENTICATION SMTP
SMTP_HOST=smtp.gmail.com
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=your_email
SMTP_PASS=your_app_password
SMTP_FROM="Kairo <your_email>"

# API KEY AI
RIWI_IA_KEY=your_ia_key
```

<Warning>
`FRONTEND_URL` must exactly match the origin where you serve the frontend. In development this is `http://127.0.0.1:5500/frontend`. A mismatch causes the Node.js CORS middleware to block all browser requests.
</Warning>

</Step>

<Step title="Configure backend-python environment">

Create the file `backend-python/.env` with the following variables:

```env
# PYTHON CONFIGURATION
PORT=8000
ENV=development

# OPENAI (used with Groq-compatible endpoint)
OPENAI_API_KEY=your_api_key

# MODEL IA
MODEL_NAME=your_model

# SUPABASE CONFIG
SUPABASE_URL=your_supabase_url
SUPABASE_SERVICE_KEY=your_service_key

# BACKEND NODE PORT
NODE_URL=http://localhost:3000

# FRONTEND PORT
FRONTEND_URL=http://localhost:5500
```

<Note>
`OPENAI_API_KEY` is the variable name used by the Groq SDK. Set it to your [Groq API key](https://console.groq.com).
</Note>

</Step>

<Step title="Install Node.js dependencies">

```bash
cd backend-node
npm install
```

This installs Express 5, Passport (GitHub + Google OAuth), `connect-pg-simple` for session storage, `@supabase/supabase-js`, and all other runtime dependencies listed in `package.json`.

</Step>

<Step title="Install Python dependencies">

Open a separate terminal and set up the Python virtual environment:

<CodeGroup>

```bash Linux / macOS
cd backend-python
python3 -m venv env
source env/bin/activate
pip install -r requirements.txt
```

```powershell Windows
cd backend-python
python -m venv env
env\Scripts\Activate.ps1
pip install -r requirements.txt
```

</CodeGroup>

</Step>

<Step title="Start the Node.js gateway">

From `backend-node`, run:

```bash
npm run dev
```

You should see the following startup banner in your terminal:

```
------------------------------------------------------------
🚀 KAIRO API GATEWAY STARTED SUCCESSFULLY
------------------------------------------------------------
📡 URL      : http://localhost:3000
🌐 Origins  : http://127.0.0.1:5500/frontend
🛠️  ENV      : development
------------------------------------------------------------
```

Verify the gateway is healthy:

```bash
curl http://localhost:3000/api/health
```

Expected response: `{"status":"active", ...}`

</Step>

<Step title="Start the Python AI microservice">

From `backend-python` (with the virtual environment activated):

```bash
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

Look for `INFO: Supabase client initialized` in the startup logs. The FastAPI interactive docs are available at `http://localhost:8000/docs`.

Verify:

```bash
curl http://localhost:8000/health
```

Expected response: `{"status":"online","service":"Kairo AI Engine", ...}`

</Step>

<Step title="Serve the frontend">

Open the `frontend/` directory using one of these methods:

**Recommended:** Install the [Live Server extension](https://marketplace.visualstudio.com/items?itemName=ritwickdey.LiveServer) for VS Code, then click **Go Live** from the status bar. This serves the frontend on port 5500.

**Alternative:** From the repository root:

```bash
npx serve -p 5500
```

Open `http://127.0.0.1:5500/frontend` in your browser.

</Step>

</Steps>

---

## Verification checklist

| Service | URL | Expected status |
|---|---|---|
| Node.js gateway | `http://localhost:3000/api/health` | `{"status":"active"}` |
| Python AI microservice | `http://localhost:8000/health` | `{"status":"online"}` |
| Frontend | `http://127.0.0.1:5500/frontend` | Login page loads |
