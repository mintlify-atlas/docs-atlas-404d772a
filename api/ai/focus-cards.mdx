## Overview

`POST /generate-focus-cards` generates exactly **6 focus cards** — 2 High / 2 Medium / 2 Low priority — personalized for a coder based on their soft-skills profile, the current module, and any topics they have been struggling with. Cards are displayed on the coder dashboard as quick-access study guides with theory, a practice task, and pro tips.

<Warning>
  This endpoint calls Groq with `max_tokens=2048`. Expect **3–8 seconds** of latency. Do not call it more than once per session; cache the response on the frontend.
</Warning>

---

## Endpoint

```
POST /generate-focus-cards
```

---

## Request body

<ParamField body="coder_id" type="integer" required>
  The coder's user ID. Used to fetch the soft-skills diagnostic (learning style, problem-solving score, autonomy score) from Supabase.
</ParamField>

<ParamField body="module_id" type="integer" required>
  The active module ID. Used to fetch the module name and up to 8 topic names for context.
</ParamField>

<ParamField body="struggling_topics" type="array" default="[]">
  List of topic name strings the coder has flagged as difficult. The two **High** priority cards are required to cover topics from this list. If empty, all cards are distributed across the module's general topics.

  ```json
  ["GROUP BY", "window functions"]
  ```
</ParamField>

---

## AI behavior

<Note>
  **Model**: `llama-3.3-70b-versatile` (configurable via `MODEL_NAME` env var), `temperature=0.7`, `max_tokens=2048`.

  The prompt injects:
  - The coder's `learning_style` (visual / kinesthetic / reading / auditory / mixed) — used to adapt the `theory` and `practice.description` tone
  - `problem_solving` and `autonomy` scores (1–5) — lower scores result in more guided practice steps
  - The module's topic list (first 8 topics) and struggling topics

  The model is instructed to return **only** valid JSON — no markdown, no backticks. If the response contains markdown fences, the router strips them before parsing.

  The required distribution is **2 HIGH / 2 MEDIUM / 2 LOW** regardless of how many struggling topics are provided.
</Note>

## Example request

<CodeGroup>

```bash cURL
curl -X POST https://your-ai-service/generate-focus-cards \
  -H "Content-Type: application/json" \
  -d '{
    "coder_id": 42,
    "module_id": 3,
    "struggling_topics": ["GROUP BY", "window functions"]
  }'
```

```python Python (httpx)
import httpx

response = httpx.post(
    "https://your-ai-service/generate-focus-cards",
    json={
        "coder_id": 42,
        "module_id": 3,
        "struggling_topics": ["GROUP BY", "window functions"],
    },
    timeout=20.0,
)
data = response.json()
cards = data["data"]["cards"]
```

</CodeGroup>

## Response

<ResponseField name="success" type="boolean">
  `true` when the cards were generated and parsed successfully.
</ResponseField>

<ResponseField name="data" type="object">
  Wrapper object returned directly from the LLM.

  <Expandable title="data fields">
    <ResponseField name="cards" type="array">
      Array of exactly 6 card objects.

      <Expandable title="card object">
        <ResponseField name="priority" type="string">
          `"high"` | `"medium"` | `"low"`. Distribution is always 2 / 2 / 2.
        </ResponseField>

        <ResponseField name="topic" type="string">
          The exact topic name from the module (e.g. `"GROUP BY"`).
        </ResponseField>

        <ResponseField name="title" type="string">
          Actionable card title (e.g. `"Domina GROUP BY con agregaciones reales"`).
        </ResponseField>

        <ResponseField name="theory" type="string">
          3–4 sentence explanation adapted to the coder's learning style.
        </ResponseField>

        <ResponseField name="practice" type="object">
          A 20-minute hands-on practice task.

          <Expandable title="practice fields">
            <ResponseField name="description" type="string">
              What the coder should do in 20 minutes.
            </ResponseField>
            <ResponseField name="example" type="string">
              Code snippet or concrete step-by-step example.
            </ResponseField>
          </Expandable>
        </ResponseField>

        <ResponseField name="pro_tips" type="array">
          Two concrete tips as strings.
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>
